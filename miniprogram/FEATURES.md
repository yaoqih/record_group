# RecordFlow 微信小程序功能说明

## 1. 产品定位

RecordFlow 是一个录音转写工具。用户上传录音或视频后，系统会自动处理音频、调用语音识别服务生成文本，并提供逐句回听、编辑和导出能力。

## 2. 用户流程

```text
阅读并同意协议
        ↓
微信快捷登录
        ↓
选择一个或多个音频/视频文件
        ↓
上传并创建任务
        ↓
查看预估时长和点数
        ↓
确认开始转写
        ↓
等待处理并查看结果
        ↓
逐句回听、导出或删除任务
```

## 3. 功能清单

### 3.1 微信登录

- 使用微信 `wx.login` 获取登录凭证。
- 服务端通过微信 `code2Session` 创建或识别用户。
- 登录前必须同意当前版本的《用户协议与隐私说明》。
- 登录成功后保存会话 token、用户资料和点数余额。
- 会话失效时，小程序会清理本地登录状态并提示重新登录。

### 3.2 用户协议与隐私说明

- 登录和上传前必须阅读并同意当前版本协议。
- 协议说明录音、转写文本、任务元数据、支付信息和日志的处理方式。
- 用户可以在“我的”页面重新查看协议。
- 撤回当前版本同意后，不能继续登录或上传新录音。

### 3.3 文件上传

- 支持选择多个音频或视频文件并加入上传队列。
- 上传前校验文件类型、文件大小和可读性。
- 显示当前文件、队列位置、上传进度和成功/失败结果。
- 上传完成后创建待确认任务，不会立即扣除点数。
- 网络短暂异常时保留当前任务状态，并自动轮询同步。

### 3.4 转写任务

任务状态包括：

| 状态 | 含义 |
| --- | --- |
| `uploaded` | 文件已上传，等待用户确认开始 |
| `starting` | 已确认并扣点，准备处理 |
| `queued` | 已进入处理队列 |
| `transcribing` | 正在进行语音转写 |
| `completed` | 转写完成，可查看和导出 |
| `failed` | 处理失败，已记录错误信息 |
| `expired` | 上传文件过期或无法继续处理 |

- 任务列表支持查看全部、处理中和已完成任务。
- 处理中任务会自动轮询状态，页面隐藏时停止轮询。
- 下拉刷新可以主动同步任务状态。
- 任务详情显示文件名、音频时长、点数费用、文本段落和保留期限。

### 3.5 点数与计费

- 上传完成后先展示预估时长和预计点数。
- 用户点击“确认开始”后才扣除点数。
- 点数余额不足时不能开始转写。
- 转写链路失败或任务过期时，服务端自动返还本次扣除的点数。
- 退款采用幂等流水，重复处理不会重复返还。
- “我的”页面支持查看余额和刷新账户状态。

### 3.6 点数充值

- 支持固定套餐：100、500、1000 点。
- 支持自定义充值数量，范围为 10-10000 点。
- 使用微信小程序虚拟支付完成充值。
- 支付完成后由服务端确认订单并增加点数。
- 取消支付、支付异常或确认延迟时，会显示对应提示，可稍后刷新余额。

### 3.7 转写结果与回听

- 完成任务后显示分句文本。
- 点击句子可以跳转到录音对应时间播放。
- 支持播放、暂停、拖动进度和查看当前播放时间。
- 转写结果可能存在错字、漏字、断句、说话人或时间戳误差，重要内容需要人工复核。

### 3.8 导出

已完成任务支持导出：

- TXT 纯文本
- SRT 字幕
- Word 文档

导出文件会先下载到微信临时目录，再打开或分享给用户。导出前应确认任务已完成，并在数据保留期限内完成备份。

### 3.9 任务删除

- 用户可以在任务详情页删除任务。
- 处理中任务不能删除，需等待处理完成、失败或过期。
- 删除会清理任务记录、关联媒体记录和已存储媒体文件。
- 删除不可恢复；队列、缓存、备份和第三方服务可能存在合理延迟。

### 3.10 账户管理

“我的”页面提供：

- 昵称和账户标识查看
- 复制用户 ID
- 同步账户和点数余额
- 查看用户协议与隐私说明
- 微信小程序虚拟支付充值
- 退出登录

## 4. 重要异常处理

- `401/403`：登录失效或会话无效，清理本地 token 后重新登录。
- `409`：任务正在处理中，暂时不能删除或执行冲突操作。
- 上传失败：保留失败项状态，可以重新选择文件上传。
- 转写失败：任务标记为失败，服务端自动退款。
- 服务端或第三方服务异常：页面显示错误信息，任务列表可通过刷新或自动轮询恢复状态。

## 5. 页面结构

| 页面 | 路径 | 职责 |
| --- | --- | --- |
| 上传首页 | `pages/index/index` | 登录、协议确认、选择文件、上传和最近任务 |
| 任务列表 | `pages/tasks/tasks` | 查看任务、筛选状态、刷新和轮询 |
| 任务详情 | `pages/task/task` | 查看状态、播放录音、查看文本、导出和删除 |
| 我的 | `pages/mine/mine` | 用户资料、点数、充值、协议和退出登录 |
| 协议 | `pages/agreement/agreement` | 查看、同意或撤回协议 |

## 6. 环境与上线配置

- 开发版和体验版默认使用测试 API：`https://test-record.blenet.top`。
- 正式版使用正式 API：`https://record-api.blenet.top`。
- 微信公众平台需要配置 `request`、`uploadFile`、`downloadFile` 合法域名。
- 真机登录必须使用真实小程序 AppID，并关闭 `USE_DEV_LOGIN`。
- 首次安装或更新依赖后，在微信开发者工具执行“工具 -> 构建 npm”。

## 7. 数据保留提示

任务音频、转写文本及关联结果原则上保留 7 天。用户应及时导出重要结果，并确认导出文件已妥善保存。
